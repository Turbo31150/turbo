# WORKFLOW_AUTO

Ce fichier décrit les protocoles de démarrage et les procédures de restauration après une compaction de mémoire. Il doit être lu à chaque réinitialisation du contexte afin de réinitialiser les états internes, les mémoires et les configurations essentielles.

- Lire ce fichier immédiatement après un reset.
- Vérifier que tous les fichiers de mémoire daily (memory/YYYY-MM-DD.md) sont lus.
- Mettre à jour les mémoires si nécessaire.
- Confirmer que les agents IA sont opérationnels.
